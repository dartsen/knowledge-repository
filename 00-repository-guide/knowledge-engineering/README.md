# Knowledge Engineering

Knowledge engineering turns source material and practitioner experience into trustworthy, findable, maintainable knowledge products. It combines information architecture, content design, metadata, provenance, validation, lifecycle governance, search/retrieval, and feedback. A search index or AI assistant is a delivery channel, not the canonical source.

## Use This Family For

- Defining a taxonomy, metadata profile, ownership, and canonical status
- Converting notes or source evidence into durable guidance
- Designing ingestion, validation, publication, indexing, and retirement flows
- Governing provenance, classification, access, retention, and reuse rights
- Operating search, retrieval, link health, review cycles, and feedback
- Supporting grounded AI without weakening source authority or permissions

## Lifecycle

```text
Authoritative sources and practitioner evidence
 -> intake, classify, and record provenance
 -> synthesize into a canonical, purpose-specific document
 -> review content, metadata, links, safety, and duplication
 -> publish and derive navigation/search/retrieval indexes
 -> observe use and feedback
 -> revise, supersede, archive, quarantine, or remove under policy
```

## Definition of Done

A canonical knowledge product has a defined audience and task, explicit scope, accountable owner, trustworthy and traceable sources, current review state, validated metadata and links, a single authority, safe handling, findable navigation terms, operational feedback, and a documented replacement/retirement path.
