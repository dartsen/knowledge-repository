# Architecture

This folder contains the Phase 2 expanded architecture-domain guides for the Git knowledge repository.

## Topics

| Topic | Purpose |
|---|---|
| [Enterprise Architecture](./enterprise-architecture/README.md) | Enterprise Architecture is the practice of aligning business strategy, capabilities, processes, data, applications, technology, security, and governance so the organization can make better long-term decisions. |
| [Solution Architecture](./solution-architecture/README.md) | Solution Architecture translates a specific business problem into a practical, buildable technical design that fits enterprise standards and can be delivered by teams. |
| [Data Architecture](./data-architecture/README.md) | Data Architecture defines how data is sourced, stored, transformed, governed, secured, and consumed across an organization. |
| [Integration Architecture](./integration-architecture/README.md) | Integration Architecture defines how systems exchange data, events, requests, and responses reliably and securely. |
| [Medallion Architecture](./medallion-architecture/README.md) | Medallion Architecture is a layered data design pattern that organizes data into progressively refined layers, commonly raw, conformed, and curated. |
| [Reference Patterns](./reference-patterns/README.md) | Reference Patterns are reusable architecture designs for common problems, allowing teams to move faster while staying consistent and governed. |

## How to Use This Domain

Use these guides to:

- Build foundational architecture vocabulary.
- Understand how architecture connects to business value.
- Document current-state and target-state designs.
- Create practical architecture decision records.
- Improve governance, troubleshooting, and operational readiness.
- Develop from beginner to professional and architecture-lead capability.
