# Enterprise Architecture

## Scope

Use this domain to connect business capabilities and quality attributes to solution, integration, information, and technology decisions.

## Recommended Starting Point

Begin with enterprise architecture principles, then move through solution architecture, integration architecture, and reusable reference patterns. Data-specific architecture belongs in Data Engineering; workload-specific implementation belongs in its technology domain.

## Key Concepts

- Business capabilities, value streams, and operating models
- Current-state, transition-state, and target-state architecture
- Architecture principles, quality attributes, and constraints
- Solution boundaries, dependencies, and tradeoffs
- Integration contracts and failure behavior
- Architecture decisions and governance

## Common Uses

- Frame a new initiative before selecting technology.
- Compare solution options and document tradeoffs.
- Design reliable system-to-system interactions.
- Prepare architecture reviews and decision records.
- Identify reusable patterns and non-functional requirements.

## Learning Sequence

```text
Enterprise context → Solution architecture → Integration architecture
→ Reference patterns → Governance and review
```
