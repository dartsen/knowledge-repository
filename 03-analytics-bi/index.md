# Analytics and BI

This folder contains Phase 3 expanded Analytics and BI guides for the Git knowledge repository.

## Topics

| Topic | Purpose |
|---|---|
| [Power BI](./power-bi/README.md) | Power BI is Microsoft’s business intelligence platform for creating semantic models, reports, dashboards, apps, and governed analytics experiences. |
| [Semantic Modeling](./semantic-modeling/README.md) | Semantic Modeling is the practice of creating a business-friendly analytical layer that defines tables, relationships, measures, calculations, hierarchies, and business terms for reporting. |
| [Dashboards](./dashboards/README.md) | Dashboards are compact visual displays that highlight key metrics, trends, exceptions, and operational signals for quick decision-making. |
| [Reporting Governance](./reporting-governance/README.md) | Reporting Governance defines how reports, dashboards, semantic models, metrics, ownership, access, certification, lifecycle, and quality are managed. |
| [Databricks to Power BI](./databricks-to-power-bi/README.md) | Databricks to Power BI is the pattern of connecting governed lakehouse data from Databricks to Power BI semantic models, reports, and dashboards. |
| [Migration Patterns](./migration-patterns/README.md) | Migration Patterns are repeatable approaches for moving reports, semantic models, data sources, dashboards, and business logic from one analytics environment or source to another. |

## Domain-Level Flow

```text
Business Question
  → Trusted Data Source
  → Semantic Model
  → Report / Dashboard
  → App / Distribution
  → Governance
  → Usage Monitoring
  → Continuous Improvement
```

## How to Use This Domain

Use these guides to:

- Build analytics and BI fundamentals.
- Understand how reports connect to trusted data and semantic models.
- Improve dashboard design and metric trust.
- Govern report lifecycle, access, and ownership.
- Support Databricks-to-Power BI migration.
- Develop from beginner to professional BI practitioner and analytics lead.
