# Power BI Beginner-to-Pro Learning Path

## Goal

Build practical mastery of **Power BI** from basic understanding to enterprise-level analytics and BI capability.

## Learning Levels

| Level | What to Learn | What to Practice | Output Artifact |
|---|---|---|---|
| Beginner | Vocabulary and purpose. | Explain the topic simply. | One-page summary. |
| Advanced Beginner | Common BI flows, tools, and stakeholders. | Map source-to-report flow. | Context diagram. |
| Intermediate Practitioner | Modeling, report design, security, and validation. | Build a simple governed report or model. | Working report/model. |
| Advanced Practitioner | Performance, lifecycle, troubleshooting, and governance. | Optimize and document BI content. | Production-ready report package. |
| Enterprise Professional | Workspace strategy, certification, metrics, and adoption. | Define BI standards for a team. | BI governance checklist or operating model. |
| Architect / Strategic Lead | Analytics strategy, migration, platform alignment. | Lead BI design or migration review. | BI reference architecture or roadmap. |

## 30-Day Plan

- Week 1: Learn vocabulary and key concepts.
- Week 2: Study source-to-model-to-report flow.
- Week 3: Build or document a simple report example.
- Week 4: Add governance, testing, and troubleshooting notes.

## 60-Day Plan

- Apply the topic to a realistic business scenario.
- Define metrics and source validation.
- Create quick reference and best practices.
- Document security and deployment approach.
- Review with a peer or business stakeholder.

## 90-Day Plan

- Build a reusable pattern or template.
- Add lifecycle and governance standards.
- Review usage and support considerations.
- Create meeting talking points.
- Add lessons learned to the repository.

## Practice Projects

1. Inventory an existing report and identify owner, source, model, and audience.
2. Create a simple semantic model with documented measures.
3. Build a dashboard around three business KPIs.
4. Create a reporting governance checklist.
5. Create a migration plan for one report.
6. Create a report rationalization decision matrix.
