# Analytics and Business Intelligence

## Scope

Use this domain for business questions, metric definitions, semantic models, Power BI, dashboard design, governed reporting, and analytics migration.

## Recommended Starting Point

Begin with the decision and metric, then define grain and semantic meaning before designing a dashboard. Continue with Power BI modeling, performance, governance, deployment, and adoption.

## Key Concepts

- Business questions, decisions, KPIs, and guardrails
- Metric definitions and semantic consistency
- Star schemas and semantic models
- DAX, Power Query, and refresh behavior
- Dashboard information design and accessibility
- Reporting governance, certification, and ownership
- Performance, capacity, deployment, and monitoring

## Common Uses

- Define a KPI or scorecard.
- Design or review a semantic model.
- Build a governed Power BI solution.
- Diagnose incorrect totals or slow reports.
- Plan a reporting-platform migration.

## Learning Sequence

```text
Business question → Metric definition → Data model → Semantic model
→ Visualization → Governance → Operations and adoption
```
