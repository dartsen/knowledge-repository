Use these pattern groups to design reliable, scalable, secure, and supportable flows without searching through one oversized reference.

## Guides

- [Power Automate Pattern Foundations](pattern-foundations.md) - TRACE, flow anatomy, and the standard outcome model
- [Power Automate Reliability Patterns](reliability-patterns.md) - selective triggers, validation, error handling, retry, idempotency, and correlation
- [Power Automate Configuration and Composition Patterns](configuration-and-composition-patterns.md) - configuration, identity, child flows, and API wrappers
- [Power Automate Scale and Orchestration Patterns](scale-and-orchestration-patterns.md) - pagination, batching, queues, watermarks, approvals, and compensation
- [Power Automate Operations, Security, and AI Patterns](operations-security-and-ai-patterns.md) - telemetry, alerting, secure data, feature flags, workers, desktop flows, and human review
- [Power Automate Standards and Reference](standards-and-reference.md) - libraries, naming, expressions, transformations, files, notifications, ALM, testing, and troubleshooting
- [Power Automate Pattern Templates](pattern-templates.md) - reusable flow-design and child-flow contract templates

## How to use this section

Read the guides in order for a structured path, or open the guide that matches the current design, delivery, or support task. The split preserves the complete source material while keeping this landing page easy to scan.
