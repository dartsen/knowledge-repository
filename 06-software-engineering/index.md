# Software Engineering

This folder contains expanded beginner-to-pro guides for the **Software Engineering** domain.

## Topics

| Topic | Purpose |
|---|---|
| [Git](./git/README.md) | Git is a distributed version control system used to track changes, collaborate safely, branch work, merge updates, and recover prior versions. |
| [GitHub](./github/README.md) | GitHub is a collaborative platform for Git repositories, pull requests, issues, code review, automation, documentation, and team workflows. |
| [Azure DevOps](./azure-devops/README.md) | Azure DevOps is Microsoft’s platform for work tracking, repositories, pull requests, pipelines, test plans, artifacts, and delivery management. |
| [CI/CD](./ci-cd/README.md) | CI/CD is the practice of automatically validating, building, testing, and deploying changes through controlled delivery pipelines. |
| [Testing](./testing/README.md) | Testing is the discipline of validating that code, data, workflows, and systems behave correctly, safely, and reliably. |
| [Code Review](./code-review/README.md) | Code Review is the process of examining changes before merge to improve quality, maintainability, security, and shared understanding. |
| [Release Management](./release-management/README.md) | Release Management controls how changes move into production through planning, approvals, deployment, communication, and rollback readiness. |
| [Repo Standards](./repo-standards/README.md) | Repo Standards define how repositories are organized, named, documented, reviewed, secured, and maintained. |
