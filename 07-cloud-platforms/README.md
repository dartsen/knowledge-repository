# Cloud Platforms

## Scope

Use this domain for Azure service selection, cloud workload architecture, resource organization, networking boundaries, managed services, scalability, and platform-specific design decisions.

## Recommended Starting Point

Start with workload requirements and quality attributes, not a service catalog. Use architecture tradeoffs to select services, then apply security, reliability, operations, performance, and cost controls.

## Key Concepts

- Subscriptions, management groups, resource groups, and landing zones
- Compute, storage, integration, data, and managed services
- Regions, zones, scaling, backup, and disaster recovery
- Network segmentation, private connectivity, identity, and secrets
- Service limits, quotas, performance, and capacity
- Monitoring, deployment, policy, and cost management

## Common Uses

- Select an Azure service for a workload requirement.
- Design a secure and supportable cloud workload.
- Compare availability and disaster-recovery patterns.
- Plan cloud governance, capacity, and cost controls.
