# DevOps

## Scope

Use this domain for source control, CI/CD, release management, infrastructure automation, containers, platform engineering, reliability engineering, observability, and production operations.

## Recommended Starting Point

Learn Git and pull-request practices, then delivery pipelines, artifact promotion, environment management, platform engineering, reliability, observability, and incident response.

## Key Concepts

- Git, GitHub, Azure DevOps, branches, and pull requests
- Continuous integration, delivery, and release controls
- Containers, infrastructure as code, and immutable artifacts
- Internal developer platforms and golden paths
- SLOs, resilience, disaster recovery, and operational readiness
- Logs, metrics, traces, alerting, and incident response

## Common Uses

- Establish a repository and branching standard.
- Build a governed delivery pipeline.
- Create a self-service platform capability.
- Define service objectives and operational readiness.
- Diagnose a distributed production failure.

## Learning Sequence

```text
Git → Pull requests → CI/CD → Artifacts and environments
→ Platform engineering → Reliability → Observability → Incident management
```
