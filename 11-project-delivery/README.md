# Project Delivery

## Scope

Use this domain for discovery, requirements, architecture and solution reviews, decision records, test strategy, deployment planning, operational readiness, support transition, retrospectives, and organization-specific implementation evidence.

## Delivery Flow

```text
Discovery → Requirements and constraints → Architecture and decisions
→ Build and test → Deployment readiness → Support transition → Learning
```

## Organization-Specific Content

Documents under `organization-specific/` describe a particular environment, portfolio, or delivery context. They remain public in this repository but must use `scope: Organization-Specific`, avoid credentials and personal data, and state what evidence was and was not reviewed.
