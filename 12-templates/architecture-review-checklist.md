# Architecture Review Checklist

## Business Alignment

- [ ] Business problem is clear.
- [ ] Business capability is identified.
- [ ] Stakeholders are known.
- [ ] Success measure is defined.

## Current State

- [ ] Current process documented.
- [ ] Current systems documented.
- [ ] Data sources identified.
- [ ] Pain points documented.

## Target State

- [ ] Target design documented.
- [ ] Options considered.
- [ ] Recommendation documented.
- [ ] Tradeoffs documented.

## Governance

- [ ] Ownership defined.
- [ ] Access reviewed.
- [ ] Security reviewed.
- [ ] Auditability considered.
- [ ] Change process identified.

## Operations

- [ ] Monitoring defined.
- [ ] Support owner identified.
- [ ] Runbook required.
- [ ] Failure handling defined.
- [ ] Post-go-live review planned.
