# Power BI Quick Reference

## Plain-English Explanation

Power BI is Microsoft’s business intelligence platform for creating semantic models, reports, dashboards, apps, and governed analytics experiences.

## Simple Analogy

Think of Power BI as the storefront for business data. The data platform prepares the ingredients, the semantic model organizes them, and Power BI presents the finished meal to decision-makers.

## Common Flow

```text
Data Sources → Power Query / Dataflows → Semantic Model → Reports → Dashboards / Apps → Users → Usage Monitoring → Improvement
```

## Why It Matters

- Self-Service Analytics
- Executive Visibility
- Operational Reporting
- Governed Insights
- Business Decision Support
- Data Storytelling

## Core Concepts

| Concept | Simple Explanation | Why It Matters | Example |
|---|---|---|---|
| Semantic Model | A reusable model that organizes data for reporting and analysis. | Prevents every report from redefining business logic. | A certified sales or policy model used by many reports. |
| Report | A multi-page interactive analytical view built from a semantic model. | Helps users explore and understand data. | Monthly operations report. |
| Dashboard | A single-page monitoring view made of pinned tiles. | Gives quick visibility into key metrics. | Executive KPI dashboard. |
| Workspace | A collaborative container for Power BI content. | Supports ownership, permissions, and lifecycle management. | Finance Analytics workspace. |
| App | A packaged experience for distributing content to users. | Improves controlled sharing. | Operations Reporting app. |
| Gateway | A bridge between Power BI cloud and on-premises data sources. | Supports secure refresh of non-cloud data. | SQL Server gateway for scheduled refresh. |
| Deployment Pipeline | A lifecycle tool for moving content across development, test, and production. | Supports controlled release management. | Dev → Test → Prod Power BI deployment. |

## Red Flags

- No business owner.
- No metric definition.
- No source validation.
- No semantic model owner.
- No security testing.
- No usage monitoring.
- Reports duplicate existing content.
- Users disagree on the numbers.
- Report refresh fails silently.
- Content is published without lifecycle control.

## Questions to Ask

1. What decision does this support?
2. Who is the audience?
3. What are the agreed metric definitions?
4. What is the trusted source?
5. Is a shared semantic model available?
6. What access controls apply?
7. How will this be deployed?
8. How will usage be monitored?
9. Who owns maintenance?
10. What should be retired or consolidated?
