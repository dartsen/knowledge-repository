# Knowledge Engineering Learning Path

| Stage | Focus | Practice artifact |
|---|---|---|
| 1. Purpose | Reader tasks, content types, authority, lifecycle | Knowledge-product brief |
| 2. Structure | Information architecture, taxonomy, metadata, stable identity | Repository and metadata map |
| 3. Evidence | Primary sources, provenance, synthesis, conflict and applicability | Source and decision record |
| 4. Delivery | Canonical authoring, review, validation, navigation, examples | Published product with quality evidence |
| 5. Retrieval | Full-text/semantic/graph choices, permissions, stable citations, evaluation | Versioned retrieval test set |
| 6. Operations | Ownership, feedback, freshness, indexing, incidents, recovery, retirement | Runbook and lifecycle scorecard |
| 7. Leadership | Governance, platform capabilities, adoption, value, continuous improvement | Knowledge-system roadmap |

Read the [canonical knowledge-engineering family](../../00-repository-guide/knowledge-engineering/README.md) alongside the repository's [metadata](../../00-repository-guide/metadata-standards.md), [lifecycle](../../00-repository-guide/content-lifecycle.md), and [new-content](../../00-repository-guide/adding-new-content.md) rules.

Complete the [canonical family example](../../15-examples/reference/canonical-knowledge-family.md). Inject a stale source, contradictory note, duplicate authority, broken link, missing owner, withdrawn content still in an index, and an unauthorized retrieval result. Demonstrate safe disposition and recovery.

Completion means you can produce traceable canonical knowledge, operate its lifecycle and derived indexes, measure reader success, and use semantic or AI capabilities without confusing them with authority.
