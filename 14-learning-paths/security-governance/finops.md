# FinOps Learning Path

| Stage | Focus | Practice artifact |
|---|---|---|
| 1. Value | Business outcomes, personas, technology economics, Inform/Optimize/Operate | Scope charter and stakeholder map |
| 2. Data | Billing periods, cost bases, FOCUS, completeness, reconciliation | Reconciled data contract |
| 3. Accountability | Ownership, allocation, shared cost, showback/chargeback, disputes | Versioned allocation policy |
| 4. Decisions | Budget, forecast, anomaly, unit economics, KPI, confidence | Decision-ready scorecard |
| 5. Optimization | Usage, rate, architecture, commitments, licenses, trade-offs | Prioritized value backlog |
| 6. Operations | Cadence, incident response, controlled automation, evidence | Runbook and monthly review pack |
| 7. Leadership | Strategy alignment, portfolio trade-offs, maturity, adjacent disciplines | Multi-quarter operating roadmap |

Read the [canonical FinOps family](../../09-security-governance/finops/README.md) in order. Build the [cost-allocation example](../../15-examples/security-governance/technology-cost-allocation.md), introduce late corrections and missing ownership, reconcile every layer, and show how a technical action affects both unit cost and service objectives.

Completion means you can explain the financial basis of a number, trace it to source, allocate it without losing conservation, choose an action with accountable owners, and prove sustained business value without weakening required controls.
