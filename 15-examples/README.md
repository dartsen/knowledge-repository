# Worked Examples

An example must demonstrate a coherent scenario rather than repeat a template with renamed headings.

Each Active example should include:

- Problem and assumptions
- Architecture or process flow
- Implementation decisions
- Failure modes and recovery
- Security and governance controls
- Operational ownership and validation
- Links to the canonical pattern and reference material

Generic generated examples awaiting review belong in `95-review-required/`, not here.
