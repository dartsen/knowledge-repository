# Infrastructure as Code Architecture Overview

## Purpose

Show how Infrastructure as Code fits into broader enterprise architecture.

## Context

```text
Business Need
  → Infrastructure as Code
  → Systems / Data / Process / People
  → Governance
  → Operations
  → Improvement
```

## Inputs

- Business need
- Requirements
- Current-state process
- Systems or data dependencies
- Governance requirements

## Outputs

- Design
- Standard
- Checklist
- Decision record
- Runbook
- Monitoring plan

## Decision Points

| Decision | Options | Tradeoff |
|---|---|---|
| Ownership | Central, domain, shared | Balance control and context. |
| Governance | Light, moderate, strict | Balance speed and risk. |
| Delivery | Manual, automated, hybrid | Balance effort and reliability. |
