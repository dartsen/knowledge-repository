# Data Engineering

This folder contains Phase 3 expanded Data Engineering guides for the Git knowledge repository.

## Topics

| Topic | Purpose |
|---|---|
| [Databricks](../../../03-data-engineering/databricks/README.md) | Databricks is a unified data and AI platform used to build, transform, govern, analyze, and operationalize data at scale using lakehouse architecture. |
| [dbt](../../../03-data-engineering/dbt/README.md) | dbt is a transformation framework that lets teams build modular SQL models, test data assumptions, document lineage, and manage analytics engineering workflows in version control. |
| [Data Modeling](../../../03-data-engineering/data-modeling/README.md) | Data Modeling is the practice of structuring data entities, relationships, definitions, and grain so data can be trusted, queried, reused, and explained. |
| [Data Quality](../../../03-data-engineering/data-quality/README.md) | Data Quality is the discipline of checking whether data is accurate, complete, consistent, valid, timely, unique, and fit for its intended use. |
| [Data Lineage](../../../03-data-engineering/data-lineage/README.md) | Data Lineage shows where data comes from, how it changes, and where it is consumed downstream. |
| [Orchestration](../../../03-data-engineering/orchestration/README.md) | Orchestration is the coordination of data tasks, dependencies, schedules, retries, monitoring, and alerts so pipelines run reliably. |
| [Unity Catalog](../../../03-data-engineering/unity-catalog/README.md) | Unity Catalog is Databricks’ unified governance layer for managing data and AI assets, permissions, lineage, discovery, and access controls across workspaces. |
| [SQL](../../../03-data-engineering/sql/README.md) | SQL is the standard language for querying, shaping, validating, and analyzing structured data. |


## How to Use This Domain

Use these guides to:

- Build strong data engineering fundamentals.
- Understand how data moves from source to consumption.
- Improve modeling, testing, documentation, and governance.
- Support Databricks, dbt, Power BI, automation, and AI use cases.
- Develop from beginner to professional data practitioner and architecture lead.
