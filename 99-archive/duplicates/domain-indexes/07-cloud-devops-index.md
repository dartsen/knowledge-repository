# Cloud and DevOps

This folder contains expanded beginner-to-pro guides for the **Cloud and DevOps** domain.

## Topics

| Topic | Purpose |
|---|---|
| [Azure](../../../07-cloud-platforms/azure/README.md) | Azure is Microsoft’s cloud platform for infrastructure, identity, data, integration, analytics, applications, AI, and operations. |
| [Docker](../../../08-devops/docker/README.md) | Docker is a containerization platform used to package applications and dependencies into portable, repeatable runtime units. |
| [Containers](../../../08-devops/containers/README.md) | Containers are lightweight isolated runtime environments that package application code, dependencies, and configuration consistently. |
| [Service Principals](../../../09-security-governance/service-principals/README.md) | Service Principals are non-human identities used by applications, automations, and pipelines to access resources securely. |
| [Identity and Access Management](../../../09-security-governance/identity-access-management/README.md) | Identity and Access Management, or IAM, controls who or what can access systems, data, platforms, and services. |
| [Monitoring](../../../08-devops/monitoring/README.md) | Monitoring is the practice of collecting logs, metrics, traces, alerts, and health signals so teams can detect and resolve issues. |
| [Infrastructure as Code](../../../08-devops/infrastructure-as-code/README.md) | Infrastructure as Code defines cloud and infrastructure resources using version-controlled code or configuration. |
