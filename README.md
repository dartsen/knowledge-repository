# Technical Knowledge Repository

This repository is a maintained technical knowledge system for architecture, software engineering, data, analytics, automation, artificial intelligence, cloud platforms, DevOps, security, governance, and business delivery.

It is designed to answer four questions quickly:

1. Where should I start?
2. Which document is authoritative?
3. What can I reuse in a project?
4. What needs review before I rely on it?

## Content Status

| Status | Meaning |
|---|---|
| Active | Reviewed, owned, and safe to use as current guidance |
| Draft | Useful work in progress that is not yet authoritative |
| Review Required | Placement, accuracy, ownership, or publication needs a decision |
| Superseded | Replaced by a named canonical document |
| Archived | Preserved for history but excluded from active guidance |

See the complete [content lifecycle](00-repository-guide/content-lifecycle.md).

## Search

Use repository search for metadata values and tags such as `content_type: Pattern Guide`, `status: Active`, `scope: Organization-Specific`, or a technology tag. Prefer canonical documents linked from a domain README or learning path.

## Maintenance

Content changes should follow [CONTRIBUTING.md](CONTRIBUTING.md) and the [repository maintenance guide](repository-management/maintenance-guide.md). Pull requests run link, metadata, duplicate, orphan, sensitive-content, and generated-index checks. Newly added unclassified material belongs in [90-inbox](90-inbox/README.md), not in an arbitrary domain. The [reorganization report](repository-management/audits/reorganization-report.md) records the 2026 baseline accounting, validation evidence, and remaining human-review work.
