# Start Here

Choose the path that matches what you are trying to accomplish.

## I Want to Learn a Technology

Start with the [learning-path map](14-learning-paths/README.md), choose a domain, and follow its sequence from fundamentals through implementation and operations.

## I Need to Solve a Technical Problem

1. Search the relevant [domain](README.md#main-domains).
2. Check its troubleshooting and operations links.
3. Use [quick references](13-reference/README.md) for commands and decision trees.
4. Confirm that the selected document is Active before treating it as authoritative.

## I Need an Architecture Pattern

- Begin with [Enterprise Architecture](01-enterprise-architecture/README.md).
- Use [Integration Architecture](01-enterprise-architecture/integration-architecture/README.md) for system-to-system design.
- Use domain architecture links for data, automation, cloud, or AI-specific patterns.
