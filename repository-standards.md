# Repository Standards

## Naming

Use lowercase kebab-case for folders and files.

## Standard Topic Folder

```text
subject-name/
├── README.md
├── reference-guide.md
├── quick-reference.md
├── architecture-overview.md
├── business-context.md
├── best-practices.md
├── troubleshooting-guide.md
├── governance-checklist.md
├── common-frameworks.md
├── learning-path.md
└── examples/
```

## Writing Rules

- Use clear headings.
- Prefer practical examples.
- Explain business context.
- Include troubleshooting and governance.
- Include beginner-to-pro learning notes.
- Avoid unnecessary jargon.
- Keep docs useful for real work.
